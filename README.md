# Desafio de Projeto 1 - Formação Power BI Analyst - Digital Innovation One - Módulo 8 - Criando um Dashboard Gerencial para Tomada de Decisões Com Power BI 📊

Este projeto faz parte do desafio de projeto "Criando um Dashboard Gerencial para Tomada de Decisões Com Power BI" da Formação Power BI Analyst da DIO. O objetivo principal foi transformar um relatório básico em uma ferramenta de análise intuitiva, priorizando a navegabilidade e a clareza visual.

## 📑 Visão Geral do Projeto
O relatório é composto por 3 páginas focadas em diferentes níveis de detalhamento financeiro:
1.  **Sales Report:** Visão geral de vendas e métricas principais.
2.  **Report de Lucro Detalhado:** Análise profunda de margens e lucros por período e produto.
3.  **Report de Vendas Detalhado:** Matriz de dados e análise temporal de vendas brutas.

## 🚀 Principais Métricas (KPIs)
O dashboard apresenta dados consolidados de um período que abrange de setembro de 2013 a dezembro de 2014:
* **Total de Vendas:** 118,73 Mi.
* **Unidades Vendidas:** 1,13 Mi.
* **Soma de Profit (Lucro):** 16,89 Mi.
* **Soma de Discounts:** 9,21 Mi.
* **Soma de COGS (Custo de Mercadorias Vendidas):** 101,83 Mi.

## 🎨 Técnicas de Visual Data Storytelling Aplicadas
Para melhorar a experiência do usuário (UX), foram aplicados os seguintes princípios de design recomendados no desafio:

* **Posicionamento Estratégico:** Distribuição dos elementos para facilitar o fluxo de leitura.
* **Contraste:** Uso de cores para destacar informações críticas e diferenciar segmentos.
* **Proporção Áurea:** Alinhamento visual para criar um layout harmônico e balanceado.
* **Segmentação de Dados:** Implementação de filtros temporais e por categoria para análises específicas.

## 🛠️ Modificações e Funcionalidades Implementadas
De acordo com as etapas do desafio, as seguintes melhorias foram executadas:

1.  **Navegabilidade:** Criação de menus e botões de navegação para alternar entre as visões de "Segmento" e "Produto".
2.  **Gráficos de Área e Cascata:**
    * O gráfico de área foi ajustado para mostrar a tendência de "Vendas por Período" ao longo dos meses.
    * Implementação de um **Gráfico de Cascata (Waterfall)** para a "Soma de Profit por Trimestre", facilitando a visualização de ganhos acumulados.
3.  **Matriz de Descrição:** Criação de uma matriz detalhada cruzando os trimestres com os anos para uma análise tabular precisa.
4.  **Tree Map:** Utilização de Tree Map para representar a "Soma de Profit por Segmento", destacando visualmente os maiores contribuintes como o setor "Government".

## 🛠️ Tecnologias Utilizadas
* **Power BI Desktop**
* **DAX (Data Analysis Expressions)** para cálculos de medidas.
* **Power Query** para tratamento e limpeza de dados.
